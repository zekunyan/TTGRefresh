Quick Start

Three practical ways to ship refresh.

Use the default UIKit-style controls, switch load more into infinite mode, or replace the UI with built-in path effects and your own UIView.

TTG
TTGRefresh
UIKit scroll refresh toolkit
1
Default pull refresh
Small API surface, visible states, and automatic state callbacks.
Works with UITableView, UICollectionView, and plain UIScrollView.
Supports async actions that can auto-end when configured.
Programmatic triggers use the same animation and callback path.
tableView.ttg.addHeaderRefresh {
    await reloadFeed()
}

tableView.ttg.triggerRefresh()
2
Infinite load
Trigger earlier near the bottom without forcing a hard pull gesture.
Tune preload distance with preloadOffset for fast feeds.
Duplicate refresh and load-more attempts are coordinated internally.
Footer UI can remain visible enough to teach the interaction.
collectionView.ttg.addInfiniteLoad(
    preloadOffset: 180
) {
    await loadNextPage()
}
3
Path effect UI
Start from built-in motion styles, then customize text, height, and views.
20 built-in refresh and load-more path effects are available.
Use TTGRefreshPathEffectTemplate to configure copy and layout.
For full control, pass your own contentView implementation.
tableView.ttg.addPathEffectHeaderRefresh(
    style: .auroraWave
) {
    await syncTimeline()
}
