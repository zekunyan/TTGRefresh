Code architecture, layering, and runtime flow

Small public API, explicit runtime, replaceable UI.

The library keeps UIKit integration in one extension layer, isolates scroll state machines in header and footer runtimes, and lets every visual style plug in through content-view protocols.

Install on UIScrollView

The proxy creates one header and one footer, stores them as associated objects, and shares inset and activity helpers per scroll view.

scrollView.ttg

Observe scroll signals

KVO tracks contentOffset, contentSize, and bounds. Header overlay mode also listens to pan gesture translation.

KVO + pan/tap

Compute progress and state

Pull distance maps to progress, trigger thresholds, will-refresh or will-load, loading, ending, hidden, and no-more-data states.

state machine

Update replaceable UI

Content views receive state and progress callbacks. Default views, path effects, and custom UI all use the same protocol boundary.

contentView protocol

Run callback task

Sync and async closures are normalized into MainActor async work. Async actions can end refreshing automatically.

Task + MainActor

Restore layout

The inset store applies top and bottom additions independently, then restores contentInset after refresh or load-more completion.

inset restoration
Mutual exclusion ActivityCoordinator allows only one active header or footer operation per scroll view.
